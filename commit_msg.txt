fix(quota): call Token Plan remains endpoint directly (no mmx required)

The /api/minimax/quota endpoint used to shell out to the `mmx` CLI via
subprocess. If the user did not have `mmx` installed (or installed it
but did not have a Token Plan), the call silently failed and the GUI
fell back to the `minimax.plan` value in config.yaml. The README
incorrectly implied that `mmx` was a dependency of the project.

Two changes:

1) Backend (web/backend/main.py)
   - Added `_fetch_quota_via_api()` — a direct httpx call to
     `GET /v1/api/openplatform/coding_plan/remains` on the regional
     API host (api.minimax.io for global, api.minimaxi.com for CN),
     with `Authorization: Bearer <api_key>`. Mirrors what mmx-cli
     does internally, no subprocess, no external dependency.
   - Added `_fetch_quota_via_mmx()` — the original subprocess call,
     kept as a fallback for users who happen to have the CLI
     installed and want its exact output format.
   - `/api/minimax/quota` now tries the direct call first, falls
     back to the mmx CLI on failure, and returns `success: false`
     with a clear message if both fail (caller can fall back to
     the `minimax.plan` field in config.yaml).
   - The direct call also checks `base_resp.status_code` (mirrors
     mmx's behavior): a non-zero value is treated as an API-level
     error even when HTTP 200. Without this, an invalid key would
     silently produce empty `model_remains` and the GUI would show
     "API key configured" with no plan info.

2) Docs
   - README "Plan auto-detection" tip now states the direct call
     is the default path; `mmx` is mentioned as an optional
     fallback. Removes the implicit "mmx is a dependency" reading.
   - README "0.3.0 Shipped" entry for auto-detection updated to
     reflect the direct-call path.
   - CHANGELOG entry under "Fixed" documents the behavior change.

Smoke-tested with a clearly invalid key: direct call now returns
None (was returning empty data), so the GUI surfaces the auth
failure instead of silently showing "no plan info".
