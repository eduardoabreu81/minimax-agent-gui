fix(settings): wire up agent settings, system prompt, and mark Todos as demo

Three remaining stubs from the 0.3.0 release that were misleading
users with toasts that looked like saves but persisted nothing.

1) Settings Modal: agent settings (model, max_steps, workspace_dir,
   region) were a no-op on Save
   - Added `PUT /api/config/agent` endpoint with per-field validation:
     region must be 'global' or 'cn', max_steps between 1 and 1000,
     model/workspace_dir non-empty strings. The endpoint writes to
     the top-level config keys (matching the existing schema) and
     stores region under the `minimax` section.
   - Extended `handleSave()` in SettingsModal.jsx to round-trip all
     four fields after the API key save. Save is now idempotent and
     always sends the current form values.
   - Found and fixed a related bug: `/api/config` was returning
     `agent: {}` because the model/max_steps/workspace_dir values
     live at the top level of config.yaml, not under an `agent`
     key. The frontend reads `data.agent.*` so the form was always
     showing the JS-side fallbacks (default model 'MiniMax-M3',
     etc.) instead of the real configured values. Projected those
     three fields under `agent` in the response so the form now
     shows what's actually in config.yaml.

2) Settings Modal: System Prompt textarea was a stub
   - The field was always empty (no `system_prompt` key in the
     config) and the Save button did nothing with it.
   - The agent loop actually loads the prompt from
     `system_prompt.md` in the config directory. Replaced the
     textarea with a note pointing to that file, so users edit
     the right place instead of typing into a dead field.

3) Code Workspace: Todos tab was hardcoded demo data
   - The tab showed a fixed 3-item list with a note that said
     "Todos will be auto-generated by the agent in Plan mode"
     (they weren't).
   - Renamed the tab to "Todos (demo)" and updated the note to
     "Demo data — auto-generated agent todos are coming soon." The
     toggle still works on the in-memory demo items but the user
     is no longer misled about persistence.

Misc:
- `.gitignore`: added `test_*.py` (scratch one-off test scripts
  that are not part of the `tests/` suite).
- CHANGELOG: new 0.3.1 entry documenting the stub cleanup.
