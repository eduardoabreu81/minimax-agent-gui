Our auth middleware re-validates the JWT on every request and it's slow. Can you refactor it to cache the decoded token per request and add proper error handling?
Thinking · 4s
The hot path is jwt.verify() running per route. I'll decode once in the middleware, attach the payload to req.user, and wrap it in a try/catch that maps verification failures to a 401 instead of a 500.
Here's the refactor. I moved verification into a single guard and added typed error handling:
middleware/auth.ts 
export const requireAuth = async (req, res, next) => {
  const token = req.headers.authorization?.slice(7);
  if (!token) return res.status(401).end();
  try {
    req.user = await verify(token);
    next();
  } catch {
    res.status(401).json({ error: 'Invalid token' });
  }
};
This decodes the token once per request and returns a clean 401 on failure. Want me to add a short-lived in-memory cache keyed by token hash too?
